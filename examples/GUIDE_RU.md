# Руководство по тестированию сетевых сценариев

## Как запустить симулятор

```bash
./run.sh
```

После запуска откроется окно Network Simulator. Чтобы открыть проект:
**File > Open Project** — выберите нужный Case.

---

## Общие действия для каждого сценария

1. Откройте проект через **File > Open Project**
2. Нажмите зелёную кнопку **Play** (треугольник) на панели инструментов — это запустит все устройства
3. Дважды кликните на компьютер (PC) — откроется консоль
4. В консоли можно вводить команды (IP уже настроен автоматически)
5. Для проверки связи используйте команду `ping`

### Полезные команды VPCS

| Команда | Что делает |
|---------|-----------|
| `show ip` | Показать текущий IP-адрес |
| `ping 192.168.1.2` | Проверить связь с другим компьютером |
| `ip 10.0.0.1/24` | Задать IP-адрес вручную |

## Case 1: Базовый пинг (Basic Ping)

### Что в проекте
- 2 компьютера (PC1, PC2)
- 1 коммутатор (Switch)
- PC1 подключён к Switch, PC2 подключён к Switch

### Что мы проверяем
Могут ли два компьютера в одной сети обмениваться данными через коммутатор.

### Что такое коммутатор
Коммутатор (switch) — это устройство, которое соединяет компьютеры в одну локальную сеть. Он запоминает адреса подключённых устройств и отправляет данные только нужному получателю.

### Настройки сети
- PC1: `192.168.1.1/24`
- PC2: `192.168.1.2/24`
- Оба в одной подсети `192.168.1.0/24`

### Шаги для тестирования

1. Откройте проект **Case1-Basic-Ping**
2. Нажмите **Play** для запуска всех устройств
3. Дважды кликните на **PC1** — откроется консоль
4. Введите команду:
   ```
   ping 192.168.1.2
   ```
5. Вы увидите ответы от PC2:
   ```
   84 bytes from 192.168.1.2 icmp_seq=1 ttl=64 time=0.512 ms
   84 bytes from 192.168.1.2 icmp_seq=2 ttl=64 time=0.408 ms
   ```

### Ожидаемый результат
Пинг проходит успешно. Два компьютера в одной подсети могут общаться через коммутатор.

## Case 2: Изоляция подсетей (Subnet Isolation)

### Что в проекте
- 3 компьютера (PC1, PC2, PC3)
- 1 коммутатор (Switch)
- Все три подключены к одному коммутатору

### Что мы проверяем
Что компьютеры в разных подсетях не могут общаться напрямую, даже если подключены к одному коммутатору.

### Что такое подсеть
Подсеть — это группа IP-адресов, которые считаются «соседями». Компьютеры в одной подсети могут общаться напрямую. Для связи между разными подсетями нужен маршрутизатор (router).

### Настройки сети
- PC1: `192.168.1.1/24` (подсеть A)
- PC2: `192.168.1.2/24` (подсеть A)
- PC3: `192.168.2.1/24` (подсеть B — другая!)

### Шаги для тестирования

1. Откройте проект **Case2-Subnet-Isolation**
2. Нажмите **Play**
3. Откройте консоль **PC1**
4. Пингуйте PC2 (та же подсеть):
   ```
   ping 192.168.1.2
   ```
   Результат: **пинг проходит** (одна подсеть)

5. Теперь пингуйте PC3 (другая подсеть):
   ```
   ping 192.168.2.1
   ```
   Результат: **пинг НЕ проходит** (разные подсети)

### Ожидаемый результат
- PC1 → PC2: связь есть (одна подсеть)
- PC1 → PC3: связи нет (разные подсети, нет маршрутизатора)

### Почему так происходит
Коммутатор работает на канальном уровне (Layer 2) — он пересылает кадры по MAC-адресам. Но PC1 видит, что адрес `192.168.2.1` не в его подсети, и пытается отправить пакет через шлюз (gateway). Шлюза нет — пакет никуда не уходит.

## Case 3: Изоляция VLAN (VLAN Isolation)

### Что в проекте
- 3 компьютера (PC1, PC2, PC3)
- 1 коммутатор с настроенными VLAN

### Что мы проверяем
Что VLAN разделяет один физический коммутатор на несколько виртуальных сетей.

### Что такое VLAN
VLAN (Virtual LAN) — это технология, которая позволяет разделить один коммутатор на несколько изолированных сетей. Компьютеры в разных VLAN не видят друг друга, хотя подключены к одному коммутатору.

### Настройки сети
- PC1: `10.0.10.1/24` — порт в VLAN 10
- PC2: `10.0.10.2/24` — порт в VLAN 10
- PC3: `10.0.10.3/24` — порт в VLAN 20

Обратите внимание: все IP-адреса в одной подсети! Но VLAN всё равно блокирует связь.

### Шаги для тестирования

1. Откройте проект **Case3-VLAN-Isolation**
2. Нажмите **Play**
3. Откройте консоль **PC1**
4. Пингуйте PC2 (тот же VLAN 10):
   ```
   ping 10.0.10.2
   ```
   Результат: **пинг проходит**

5. Пингуйте PC3 (другой VLAN 20):
   ```
   ping 10.0.10.3
   ```
   Результат: **пинг НЕ проходит**

### Ожидаемый результат
- PC1 → PC2: связь есть (оба в VLAN 10)
- PC1 → PC3: связи нет (PC3 в VLAN 20)

### Почему так происходит
Коммутатор помечает каждый порт номером VLAN. Кадры (frames) передаются только между портами с одинаковым VLAN. Это как если бы PC1/PC2 и PC3 были подключены к разным коммутаторам.

## Case 4: Хаб против Коммутатора (Hub vs Switch)

### Что в проекте
- 4 компьютера
- 1 хаб (Hub) — к нему подключены PC1 и PC2
- 1 коммутатор (Switch) — к нему подключены PC3 и PC4
- Хаб и коммутатор соединены между собой

### Что мы проверяем
Разницу в поведении хаба и коммутатора.

### Что такое хаб
Хаб (hub) — простое устройство, которое получает данные от одного компьютера и отправляет их ВСЕМ остальным подключённым устройствам. Это неэффективно и небезопасно.

### Чем коммутатор лучше хаба
Коммутатор запоминает MAC-адреса устройств и отправляет данные только нужному получателю. Это быстрее и безопаснее.

### Настройки сети
- PC1: `10.0.0.1/24` (через хаб)
- PC2: `10.0.0.2/24` (через хаб)
- PC3: `10.0.0.3/24` (через коммутатор)
- PC4: `10.0.0.4/24` (через коммутатор)

### Шаги для тестирования

1. Откройте проект **Case4-Hub-vs-Switch**
2. Нажмите **Play**
3. Откройте консоль **PC1**
4. Пингуйте все компьютеры:
   ```
   ping 10.0.0.2
   ping 10.0.0.3
   ping 10.0.0.4
   ```
   Все пинги должны пройти — все в одной подсети.

5. Для наблюдения за разницей: кликните правой кнопкой на кабель между хабом и PC2, выберите **Start capture**. Потом пингуйте с PC1 на PC3. Вы увидите, что PC2 тоже получает эти пакеты (хаб рассылает всем).

### Ожидаемый результат
- Все компьютеры могут пинговать друг друга
- Хаб рассылает трафик всем портам (broadcast)
- Коммутатор отправляет трафик только получателю (unicast)

## Case 5: Звёздная топология (Star Topology)

### Что в проекте
- 6 компьютеров (PC1–PC6)
- 1 центральный коммутатор
- Все компьютеры подключены к одному коммутатору (форма звезды)

### Что мы проверяем
Работу звёздной топологии — самой распространённой схемы подключения в локальных сетях.

### Что такое звёздная топология
Звёздная топология — это когда все устройства подключены к одному центральному узлу (коммутатору). Если один компьютер отключится, остальные продолжат работать.

### Настройки сети
- PC1: `10.0.0.1/24`
- PC2: `10.0.0.2/24`
- PC3: `10.0.0.3/24`
- PC4: `10.0.0.4/24`
- PC5: `10.0.0.5/24`
- PC6: `10.0.0.6/24`

### Шаги для тестирования

1. Откройте проект **Case5-Star-Topology**
2. Нажмите **Play**
3. Откройте консоль **PC1**
4. Проверьте связь со всеми компьютерами:
   ```
   ping 10.0.0.2
   ping 10.0.0.3
   ping 10.0.0.4
   ping 10.0.0.5
   ping 10.0.0.6
   ```
   Все пинги должны пройти.

5. Откройте консоль **PC6** и пингуйте PC1:
   ```
   ping 10.0.0.1
   ```
   Тоже работает — связь двусторонняя.

### Ожидаемый результат
Все 6 компьютеров могут общаться друг с другом через центральный коммутатор. Это демонстрирует, как работает типичная офисная сеть.

## Case 6: Несколько коммутаторов (Multi-Switch)

### Что в проекте
- 4 компьютера
- 2 коммутатора, соединённых друг с другом (uplink)
- PC1 и PC2 подключены к Switch1
- PC3 и PC4 подключены к Switch2

### Что мы проверяем
Что компьютеры могут общаться через несколько коммутаторов.

### Что такое uplink
Uplink — это соединение между двумя коммутаторами. Оно позволяет компьютерам из разных коммутаторов общаться, как если бы они были в одной сети.

### Настройки сети
- PC1: `172.16.0.1/24` (Switch1)
- PC2: `172.16.0.2/24` (Switch1)
- PC3: `172.16.0.3/24` (Switch2)
- PC4: `172.16.0.4/24` (Switch2)

### Шаги для тестирования

1. Откройте проект **Case6-Multi-Switch**
2. Нажмите **Play**
3. Откройте консоль **PC1**
4. Пингуйте компьютер на том же коммутаторе:
   ```
   ping 172.16.0.2
   ```
   Результат: **работает** (оба на Switch1)

5. Пингуйте компьютер на другом коммутаторе:
   ```
   ping 172.16.0.4
   ```
   Результат: **тоже работает** (кадры идут через uplink)

6. Откройте консоль **PC4** и пингуйте PC1:
   ```
   ping 172.16.0.1
   ```
   Результат: **работает** в обе стороны

### Ожидаемый результат
Все 4 компьютера могут общаться друг с другом, несмотря на то, что подключены к разным коммутаторам. Uplink-соединение объединяет две группы в одну сеть.

### Почему это важно
В реальных сетях используются десятки коммутаторов. Они соединяются друг с другом uplink-кабелями, создавая единую сеть для сотен компьютеров.
